Мини-приложение: свой домен / поддомен Timeweb (tw1.ru)
====================================================

Пошаговый чеклист под панель Timeweb + BotFather (скрины, SSL, .htaccess):
  см. файл BOTFATHER_AND_TIMEWEB_CHECKLIST.txt в этой папке.

---


Цель: по адресу https://ВАШ_ПОДДОМЕН/ открывается не заглушка хостинга, а тот же
сайт, что отдаёт бот (aiohttp: статика web_app/ + API /api/...).

В .env на сервере (рядом с main.py):
  WEBAPP_PUBLIC_URL=https://ВАШ_ПОДДОМЕН
без слэша в конце. Пример:
  WEBAPP_PUBLIC_URL=https://botgadamaka.cv324192.tw1.ru

WEBAPP_PORT в .env должен совпадать с портом, на который смотрит прокси (часто 8080).
После правки .env перезапустите бота (main.py).

Почему видна заглушка «Ваш аккаунт готов к работе»
-----------------------------------------------
Поддомен привязан к каталогу сайта (public_html), а не к процессу Python. Запросы
не доходят до aiohttp — нужно проксирование на порт или VPS с nginx (см. TIMEWEB.txt).

Что сделать в панели Timeweb (зависит от тарифа)
-----------------------------------------------
1) Сайты и домены → сайт поддомена → проверить «корневая папка».
   Удалите из неё заглушку index.htm / index.html, если мешает (после настройки прокси).

2) Найдите в настройках сайта один из вариантов:
   - проксирование на порт / приложение на порту / Node.js (укажите порт = WEBAPP_PORT);
   - или отдельная инструкция Timeweb «внешний порт» для приложения.

3) SSL: для поддоменов *.cv324192.tw1.ru сертификат обычно выдаётся панелью автоматически.
   В браузере должен открываться https://ВАШ_ПОДДОМЕН/index.html с интерфейсом бота
   (не страница «аккаунт готов»).

4) Убедитесь, что бот запущен и слушает 0.0.0.0:WEBAPP_PORT (как в вашем .env).

Проверка с сервера или ПК:
  curl -sI https://ВАШ_ПОДДОМЕН/index.html
Ожидается HTTP 200 и типичный HTML мини-приложения.

Telegram
--------
В @BotFather → Bot Settings → Menu Button укажите:
  https://ВАШ_ПОДДОМЕН/index.html
(тот же хост, что в WEBAPP_PUBLIC_URL.)

Кнопка «Мини-приложение» в боте берёт URL из WEBAPP_PUBLIC_URL + /index.html (см. код).

Если на тарифе нет прокси на порт
---------------------------------
Разумный путь — VPS по инструкции TIMEWEB.txt (nginx → 127.0.0.1:WEBAPP_PORT) или
уточнение в поддержке Timeweb, как привязать поддомен к long-running Python на порту.
