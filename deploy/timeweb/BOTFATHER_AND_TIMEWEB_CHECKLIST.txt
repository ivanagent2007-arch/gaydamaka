Чеклист: Timeweb BOTGADAMAKA + BotFather (мини-приложение)
========================================================

ВАЖНО: готовый .htaccess для КОРНЯ сайта BOTGADAMAKA (не папка Python):
  deploy/timeweb/for_BOTGADAMAKA_site_root/
  Там же READ_SITE_ROOT_ru.txt — по шагам.

Домен: https://botgadamaka.cv324192.tw1.ru
Цель: в браузере открывается интерфейс мини-приложения (не заглушка Timeweb),
      в Telegram — тот же URL в BotFather и в .env бота.

────────────────────────────────────────────────────────
1) Timeweb — SSL и HTTPS (на скрине сейчас HTTP)
────────────────────────────────────────────────────────
Telegram принимает только HTTPS.

  • Панель → «Домены и SSL» / «SSL-сертификаты» → выпустите бесплатный
    сертификат (Let's Encrypt) для поддомена botgadamaka.cv324192.tw1.ru
    (или включите SSL для этого сайта, если кнопка в настройках сайта).

  • Сайт BOTGADAMAKA → настройки (шестерёнка) → включите перенаправление
    на HTTPS / безопасное соединение (после того как сертификат уже выдан).

Проверка: в браузере откройте https://botgadamaka.cv324192.tw1.ru/ —
адресная строка должна оставаться с https://, без предупреждений сертификата.

────────────────────────────────────────────────────────
2) Timeweb — чтобы открывалось мини-приложение, а не заглушка
────────────────────────────────────────────────────────
Сайт сейчас привязан к папке (PHP), а бот слушает порт (aiohttp, WEBAPP_PORT).

  A) В корень сайта BOTGADAMAKA (файловый менеджер / SSH) положите файл
     .htaccess по примеру:
       deploy/timeweb/apache-proxy-to-aiohttp.htaccess.example
     Переименуйте в точности: .htaccess
     Порт внутри файла замените, если у бота в .env не 8080.

  B) Удалите или переименуйте index.htm / index.html от заглушки Timeweb,
     чтобы не перехватывали запрос раньше прокси.

  C) На сервере бот должен быть запущен (nohup python3 main.py …) и слушать
     0.0.0.0:WEBAPP_PORT (или 127.0.0.1:WEBAPP_PORT, если прокси только локально).

Проверка:
  curl -sI https://botgadamaka.cv324192.tw1.ru/index.html
Ожидается HTTP/2 или HTTP/1.1 200 и тело HTML приложения (не «аккаунт готов»).

Если .htaccess даёт 500 — в поддержку Timeweb про mod_proxy; либо VPS + nginx
(файл TIMEWEB.txt в этой же папке).

────────────────────────────────────────────────────────
3) Версия «Python 3.4» в строке сайта (скрин настроек)
────────────────────────────────────────────────────────
Это версия для связки сайта с Apache (mod_wsgi), не обязательно тот Python,
которым вы запускаете бота по SSH.

  • Проверьте по SSH:  python3 --version  (в каталоге с venv бота)
  • Если в панели для сайта доступна более новая ветка Python — выберите
    3.10+ только если пользуетесь официальной схемой Python-хостинга Timeweb.
  • Ваш текущий запуск «python3 main.py» из venv зависит от того, какой
    python3 на сервере, а не от числа «3.4» в списке PHP/Python сайта.

────────────────────────────────────────────────────────
4) Файл .env на сервере (рядом с main.py)
────────────────────────────────────────────────────────
  WEBAPP_PUBLIC_URL=https://botgadamaka.cv324192.tw1.ru
без слэша в конце. После правок — перезапуск бота.

────────────────────────────────────────────────────────
5) @BotFather (скрины: Menu Button и Main App)
────────────────────────────────────────────────────────
Сейчас в Menu Button указан старый trycloudflare.com — его нужно ЗАМЕНИТЬ.

Скопируйте один и тот же URL (со слэшем и index.html):

  https://botgadamaka.cv324192.tw1.ru/index.html

  • Bot Settings → Menu Button → вставьте URL выше, текст кнопки, например:
      Мини-приложение
    → Save

  • Mini Apps → Main App → Enter URL → вставьте тот же URL → Save
    (если раздела нет — достаточно Menu Button + кнопки в самом боте.)

После того как сайт реально отдаёт 200 на /index.html, мини-приложение
в Telegram откроется без ERR_CONNECTION_CLOSED.
